# 数据分析面试速记指南（最后2小时突击专用）

## 一、 “面试话术版”极简总结：数据清洗与数据建模

### 1. 数据清洗是干什么的？数据需要处理成什么样子？
**大白话解释**：
“实际业务中拿到的数据往往是‘脏’的，比如有缺失、有重复、格式不对、甚至有异常的错位值。数据清洗就是把这些‘脏’数据变成**干净、规整、可以直接用来分析或跑模型**的过程。”

**面试话术（背诵版）**：
> “在我的理解中，数据清洗主要包含四个步骤：
> 1. **处理缺失值**：根据业务情况，选择填充（均值/中位数/业务默认值）或者删除。
> 2. **处理重复值**：对唯一标识字段（如订单ID、用户ID）去重。
> 3. **格式转换与标准化**：比如把字符串格式的时间转成日期类型，把带有人民币符号的字符串转成纯数字等。
> 4. **处理异常值**：识别不合理的极端值（如年龄为200岁），根据业务规则剔除或修正。
> 
> **最终要处理成的样子**是：每一列数据类型正确无误、没有不合理的空值、没有重复记录的**结构化表格数据**，可以直接被后续的 BI 工具或算法模型调用。”

### 2. 数据建模主要是干什么的？
**大白话解释**：
“这里的‘建模’（通常指数仓建模，不是复杂的机器学习算法）其实就是**把数据按照业务逻辑组织起来，建几张好用的表**，让后续查数据更方便，不用每次都写几百行 SQL。”

**面试话术（背诵版）**：
> “我觉得数据建模的核心是为了**提升查询效率、降低数据冗余、并统一业务口径**。
> 常见的维度建模方法（比如星型模型）就是把数据分为**事实表**（记录发生的业务动作，比如交易流水）和**维度表**（记录环境和对象信息，比如用户属性表、商品属性表）。
> 建模就是把杂乱的原始数据，分层整理（比如从 ODS原始层 -> DWD明细层 -> DWS汇总层 -> ADS应用层），最终产出一张张能直接回答业务问题的宽表或聚合表。”

## 二、 边缘技术栈“大白话”应对口径（只需了解概念）

如果面试官问起，就用下面这些一两句话带过，表明自己“懂这是干嘛的”，同时强调自己核心能力在于 SQL 和 Python 数据处理。

- **Linux**：
  > “工作中主要使用基础的 Linux 命令进行环境维护和查看日志，比如用 `ls`, `cd`, `grep` 查文件，或者用 `tail -f` 看实时日志，以及用 `crontab` 挂一些定时的 Python 脚本。”
- **Hadoop & Hive**：
  > “了解大数据生态的基础。我知道 Hadoop 是用来做分布式存储和计算的（HDFS/MapReduce）。Hive 则是在 Hadoop 之上的数据仓库工具，它把 SQL 转换成底层的 MapReduce 任务。平时主要是写 Hive SQL（HQL）来从大数据集群里提取数据。”
- **ETL 工具 (Extract, Transform, Load)**：
  > “ETL 指的是数据的抽取、清洗转换和加载。我了解通常会有 Kettle 或 DataX 等工具来做这个事，不过在我这边的经历中，很多复杂的 Transform 逻辑我也是直接用 SQL 加上 Python (Pandas) 来实现的。”
- **FineBI (或其他 BI 工具如 Tableau, PowerBI)**：
  > “FineBI 是一种自助式的 BI 报表工具。我认为所有的 BI 工具底层逻辑都是相通的：连接清洗好的数据源，通过拖拽维度（如时间、地区）和度量（如销售额），来快速生成可视化的图表和 Dashboard。”

## 三、 SQL 面试高频核心考点梳理

面试中基础的 `SELECT ... FROM ... WHERE ... ORDER BY ... LIMIT ...` 肯定会用到，但拉开差距的是以下三点。在 `demo.py` 中有实际演示，这里做语法速记：

### 1. 多表关联 (JOIN)
- **核心场景**：把订单表和用户表连起来，看某个用户的订单情况。
- **面试最爱问**：`LEFT JOIN` 和 `INNER JOIN` 的区别。
  - `INNER JOIN`：只有两边都有的数据才会保留。
  - `LEFT JOIN`：左边表的数据全保留，右边表匹配不上的地方填 `NULL`。

### 2. 分组聚合 (GROUP BY & HAVING)
- **核心场景**：算每个部门的总薪水，或者算每个月的总销售额。
- **易错点**：`WHERE` 是分组前的过滤，`HAVING` 是分组后的过滤（比如只看总销售额 > 10000 的月份）。

### 3. 窗口函数 (Window Functions) - **面试杀手锏，极高频！**
- **核心场景**：求每个部门工资前三名的人（分组内的 Top N 问题）；求连续登录天数等。
- **必背公式**：
  ```sql
  ROW_NUMBER() OVER (PARTITION BY 分组字段 ORDER BY 排序字段 DESC)
  ```
  - **解释**：对每一组（PARTITION BY）里的数据，按照某个字段排序（ORDER BY），然后给每行发一个排名号（1, 2, 3...）。
